Engaging with the Wiphala

A Code-Generation Hardened CS1 Final

Calvin Deutschbein · Shouvik Ahmed Antu — Willamette University

20-minute talk · Q&A included

Welcome, brief context: CS151 in Python, classic Nifty assignments now easily solved by LLMs. We designed a final that keeps ACM CS1 learning outcomes while productively engaging with AI.

Q&A

What would it take for your final to survive a GPT prompt?

Seed discussion. Offer to share rubric and milestone text.

Why redesign CS1 assessment?

  • LLMs produce perfect solutions to many classic assignments
  • Students can bypass authentic practice and debugging
  • Goal: preserve core learning + teach tool literacy

Design around AI instead of banning it.

Set up the problem. Emphasize we want students to learn with tools while still building computational thinking.

Course context

  • CS151 (Python) adapted from Stanford; Nifty assignments + ImageShop
  • Primary learning via projects; milestone based
  • Observed drop: scripting → multi-file programming
Motivation for a cumulative, interdependent final that surfaces true understanding versus prompt engineering.

Design principles

  • Align to ACM CS1 objectives
  • Leverage intentional ambiguity
  • Prefer visual / image-based specifications
  • Make milestones interdependent
  • Embed cultural/ethical context
“Intentional ambiguity” is key: require students to design representations and make choices before coding.

The Wiphala (concept)

Use the screenshot as a visual anchor. In the assignment, students generate this programmatically without naming “Wiphala.”

Final structure & milestones

  1. Strings & Files
  2. Interactive Graphics
  3. Reading Python (visual spec → code)
  4. Defining Classes (image collage)
  5. Fundamental Python (vector ops)

~7–8 minutes total across the five pieces

1) Strings & Files

  • Non-standard file: Spanish color names, space-separated RGB, no headers
  • Students decide representations (lists, dicts, tuples)
  • LLMs can parse CSVs—this breaks assumptions via ambiguity
rojo 219 10 19
naranja 236 120 8
...
Emphasize design choices. Many valid solutions emerged, showing genuine understanding.

2) Interactive Graphics

  • Render the Wiphala pattern via loops / slices
  • Common AI error: simple diagonal, wrong cycle/offset
  • We intentionally avoid naming the pattern in the prompt
Students must reason about 2D indexing and modular sequences.

3) Reading Python (reimagined)

  • From tracing given code → writing code from images
  • Fixed-pixel HTML prevents easy prompt-pasting of images
  • Encourages structural reasoning, not rote output

4) Defining Classes → Collage

  1. Read image files from names in CSV
  2. Crop to square; scale uniformly
  3. Assemble collage Wiphala

LLMs misread the inputs (random tiles vs concatenated image) → brittle.

Multi-step dependency exposes shallow understanding and promotes pipeline thinking.

5) Fundamental Python (vector ops)

  • Integer luminance: r, g, b = color; (4*g + 3*r + b)//8
  • Map high luminance → rojo scale; low → black
  • Vectorized NumPy operations, not floats
Subtle implementation details confound code-gen defaults and reinforce fundamentals.

Results

  • Quasi-uniform final score distribution; clear differentiation
  • Final uncorrelated with midterm/project scores (r² ≈ 0.006)
  • Some high project scorers struggled to write code unaided
Balanced view: success in revealing understanding, but consider equity and preparation.

Implications & lessons

  • Design tasks that require interpretation, synthesis, judgment
  • Use ambiguity + interdependent milestones
  • Bring cultural context and ethics into CS1
  • Prepare students with in-class practice using similar patterns

The big picture

We don’t ban AI; we raise the bar for authentic learning.
